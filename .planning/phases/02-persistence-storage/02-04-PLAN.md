Deliver the coordinator SQLite store (STOR-03) and wire the participant store into the CLI. The DB
holds roster, ceremony transcripts, session logs, policy config, and the churn ledger — PUBLIC data
only, never encrypted (D-11) — with the roster populated by REAL npubs from the Phase 2 identity keys

migratable public store; the participant persona has a self-contained store surface; and headless CI
(the Phase 3 n=100 persist/reload check) can drive the whole thing without a TTY.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| coordinator DB → any reader | DB is public by design; a leaked secret column would be catastrophic and unencrypted |
| test/CI passphrase seam → shipped binary | The in-code PassphraseSource must never become a production passphrase path |
| store → CLI read path | share status must read only public plaintext, never trigger an unlock |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-14 | Information Disclosure | secret/nonce/partial leaked into the coordinator DB | high | mitigate | SCHEMA_V1 holds only public columns; DB never encrypted by design (D-11); review each column |
| T-02-15 | Spoofing | roster npub not the real identity key (breaks Phase 7 roster hash commit) | medium | mitigate | Roster populated from IdentityKeypair::npub() (D-15); identifier = stable hex of frost Identifier (Pitfall 16) |
| T-02-16 | Elevation | in-code test PassphraseSource reachable in the shipped binary | high | mitigate | in-code impl is test-only; production wiring is interactive-only (D-01/D-03); share status never unlocks |
| T-02-17 | Tampering / DoS | concurrent writers corrupt the DB | medium | mitigate | WAL + busy_timeout(5s); single-writer coordinator discipline |
</threat_model>
