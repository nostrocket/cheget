<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator terminal → process | Passphrase enters via no-echo stdin; must not echo, persist, or land in env/history |
| plaintext secret in RAM → disk | Secret bytes must be age-encrypted before any write and zeroized after use |
| crates.io → build | New dependency (rpassword) crosses the supply-chain boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure | store::crypto weak KDF | high | mitigate | Explicit scrypt `log_n = 18`; scrypt is the sole recipient; age (audited) not hand-rolled (RESEARCH V6) |
| T-02-02 | Information Disclosure | decrypted bytes lingering in freed memory | high | mitigate | decrypt_secret returns Zeroizing<Vec<u8>>; caller drops at point of use (D-06) |
| T-02-03 | Information Disclosure | passphrase echoed / resident | high | mitigate | No-echo read into SecretString; no env var/CLI flag ships (D-01/D-03, Pitfall 7) |
| T-02-04 | Tampering / DoS | crash mid-write corrupts/truncates a live share | high | mitigate | Atomic temp+fsync+rename + directory fsync (D-07, Pitfall 2) |
| T-02-SC | Tampering | rpassword crate install (crates.io) | high | mitigate | Blocking-human legitimacy checkpoint (Task 1) before install; other deps Approved in RESEARCH audit |
