<objective>
Build the encrypted between-round DKG checkpoint store (STOR-02) as a capability wired at the seam
(D-08): concrete `put/load` methods typed to `dkg::round1::SecretPackage` and
`dkg::round2::SecretPackage` ONLY, encrypted under the store passphrase (D-09) inside the participant
store (D-11), with wipe-on-success / keep-on-abort semantics (D-10). This is the DKG-round-secret half
of STOR-02; the nonce half is already satisfied structurally by the non-serializable `EphemeralNonces`
and must stay that way.

Purpose: the checkpoint store is the deliberate INVERSE of the nonce type — dkg SecretPackages must
survive between rounds and so are serializable and get concrete persist methods, while a signing nonce
is non-serializable and gets none. The type-restricted API makes "make signing resumable like ceremonies"
(the highest-severity key-extraction bug class) a non-expressible operation. Also lay the `#[ignore]`
n=100 persist/reload harness stub that Phase 3 will run at scale.
Output: CheckpointStore + dkg roundtrip + wipe/keep + nonce-guard tests green; n=100 harness stub present.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signing nonce ↔ persistence | A signing nonce must never be serializable or reach any store method |
| dkg round secret → disk | dkg SecretPackages checkpoint between rounds; must be encrypted at rest under the store passphrase |
| ceremony success/abort → files | Success wipes secrets; abort retains them for idempotent resume |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-10 | Information Disclosure / Elevation | signing nonce swept into a checkpoint (key extraction) | critical | mitigate | Type-restricted checkpoint API (concrete dkg SecretPackage only) + non-serializable nonce; store-side guard (Pattern 2, Pitfall 1) |
| T-02-11 | Information Disclosure | dkg round secret written unencrypted or lingering | high | mitigate | serialize→Zeroizing→encrypt_secret→atomic write under store passphrase (D-09) |
| T-02-12 | Tampering / DoS | crash mid-checkpoint corrupts a round file, breaking resume | high | mitigate | Atomic write + directory fsync (D-07); keep-on-abort for resume (D-10) |
| T-02-13 | (design) | "wiped checkpoint = secret gone" false assurance | low | accept | Deletion labeled best-effort hygiene; the sweep is the revocation (SPEC §11.1, Pitfall 9) |
</threat_model>
