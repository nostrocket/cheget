<objective>
Introduce the load-bearing architectural seam: a `Transport` trait plus an in-memory/in-process
stub. This is the seam every later ceremony phase (3–6) runs against with zero relay code, and
Phase 7 swaps in real `FileTransport`/`NostrTransport` behind the same trait with no call-site
churn (D-08). Phase 1's signing session (01-04) is its first consumer — it collects round-1
commitments and distributes the signing package over this seam (SIGN-02).

Purpose: proving the entire system locally before any network code is only possible because
message passing hides behind this trait. The envelope must be shaped for the future Nostr event
model (per-class kinds, directed vs broadcast, dedup by id) while concretely needing only the
stub now.
Output: `transport/` (trait + opaque-bytes envelope + in-memory stub) and its unit tests.
</objective>
