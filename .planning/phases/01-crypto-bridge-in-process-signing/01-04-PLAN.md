<objective>
Assemble the two-round FROST signing session over the `Transport` stub and prove the crown-jewel
outcome: an in-process 51-of-100 tweaked signature that verifies against the output key Q,
finalizes a PSBT, and confirms a key-spend on the auto-spawned regtest node. This plan wires
together the bridge (01-01), crypto core + nonces (01-02), chain backend (01-03), and transport
(01-05), and encodes the two coordinator-untrusted gates from the first signing flow:
display-before-sign (SIGN-07) and verify-against-Q (SIGN-04).

Purpose: this is the whole value proposition de-risked — DKG → address → sign → aggregate →
verify → broadcast → confirm. The gates and abort semantics (over-provision, new-session-on-abort,
no commitment reuse) must be built now so Phase 7 inherits them at scale.
Output: `session/`, the coordinator sign/aggregate/verify helpers, the sign command, and the
small-n (PR) + n=100 (nightly) confirmed-key-spend end-to-end tests plus adversarial tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| coordinator → participant (sighash) | Untrusted coordinator could hand a malicious hash; participants recompute from the PSBT |
| tweaked vs untweaked aggregation | Inconsistent tweak or `Some(merkle_root)` yields a wrong-key/invalid signature |
| session lifetime → nonce reuse | A restart/timeout that reuses commitments is the key-extraction bug class |
| aggregated signature → output key Q | Must verify against Q (on-chain key), never internal P |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-blindsign | Spoofing / Tampering | `session/display.rs` round-2 gate | critical | mitigate | Display-before-sign; each seat recomputes sighash from the PSBT; ack required unless `--yes`; blind-sign refusal test (SIGN-07) |
| T-01-tweak | Tampering | `crypto/sign.rs` aggregate/verify | high | mitigate | Single tweaked pipeline, `aggregate_with_tweak(…, None)` hard-wired; verify against Q; untweaked path never exposed (SIGN-03/04, Pitfall 7) |
| T-01-noncereuse | Information Disclosure / Elevation | session abort/timeout | critical | mitigate | New session id + fresh `EphemeralNonces` on any abort/timeout; commitments never reused; runtime rejection test + 01-02 compile-fail (SIGN-05/06) |
| T-01-dos | Denial of Service | 51-way session robustness | medium | mitigate | Over-provisioned liveness poll; new session on abort (Pitfall 11); stress at scale is Phase 7 |
| T-01-culprit | Tampering | aggregation | medium | mitigate | 3.0 cheater detection default-on; surface `Error::culprits()` on abort (SIGN-06) |
</threat_model>
