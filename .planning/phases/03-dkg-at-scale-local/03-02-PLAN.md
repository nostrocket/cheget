<objective>
Rework `sign`'s persisted-share path to load 51 of the 100 store roots produced by 03-01 and re-prove
the crown-jewel confirmed regtest key-spend from PERSISTED shares (D-05). Today `cli/sign.rs` re-runs
`run_inprocess_dkg` on every invocation because nothing was persisted to load; this plan makes `--persist`
load real KeyPackages from disk and feed the existing `SigningSession`, changing only the SOURCE of the
key material — the two-round flow, nonce lifetime, and display gate stay exactly as proven in Phase 1.

Purpose: Deliver the read/sign half of KEY-06's persist/reload-through-real-commands criterion and satisfy
ROADMAP criterion 2 (confirmed regtest key-spend from persisted shares), with small-n PR-gate correctness
plus a one-time full-100 functional smoke (D-06).
Output: A one-call per-root loader (`ParticipantStore::load_only_active`), a `pub` `load_persisted_shares`
read-glue helper that `sign::run` delegates to (discover/select-`t`/assemble), a shared loaded-shares
chain-proof helper (`run_confirmed_key_spend_from_shares`), and the `tests/persisted_sign.rs` end-to-end test
that drives the read helper into the confirmed key-spend.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk (store) -> process memory | Encrypted KeyPackage bytes are read and decrypted; plaintext must live only in `Zeroizing` and drop at use (D-06). |
| operator terminal -> process | The unlock passphrase is entered interactively (no-echo, single prompt) and reused across the 51 roots; never logged/persisted. |
| coordinator PSBT -> signer | The PSBT is untrusted input; each seat recomputes the sighash and must not blind-sign (SIGN-07). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-05 | Information Disclosure | signing nonces on the persisted-share path | high | mitigate | Only the SOURCE of `key_packages` changes; `SigningSession` still mints in-memory-only nonces (SIGN-05) and this plan adds no persistence of round secrets. Task 2 keeps `session.run` unchanged; no nonce type is serialized. |
| T-03-06 | Information Disclosure | decrypted KeyPackage plaintext during load | high | mitigate | Loads go through `ParticipantStore::load_share`, which decrypts into `Zeroizing` and drops at fn end (D-06). `load_only_active` returns only the parsed `KeyPackage`, not raw bytes. |
| T-03-07 | Spoofing/Tampering | coordinator-supplied sighash (blind signing) | high | mitigate | The display-before-sign gate is preserved: `session.preview()` + `--yes`/`prompt_ack`, and `round2` recomputes the sighash from the PSBT (SIGN-07). Task 2 must not bypass it; `inproc_sign::round2_display_gate_refuses_blind_sign` continues to cover the invariant. |
| T-03-08 | Tampering | seat-root discovery / hex->SeatId | medium | mitigate | Roots are discovered by a `seat-` prefix under a resolved base; `seat_from_hex` validates via `frost::Identifier::deserialize` and `KeyId::new` re-validates the manifest key_id (path-traversal guard T-02-12). Malformed entries return `StoreError`, not a panic. |
| T-03-09 | Denial of Service | fewer than t roots available | low | accept | Handled as a clear error (insufficient roots) before any session starts; no partial signing. |
| T-03-SC | Tampering | dependencies | n/a | accept | No new crates added — supply-chain surface unchanged; existing `Cargo.lock` pins hold. |
</threat_model>
